      Steps #

      Steps are the basic building block for jobs. These are the actual actions part of GitHub Actions. They are essentially a task that you need done. This could be a task that uses code in the repository, but it does not have to be. A step could essentially do anything.

      Think of steps as analogous to `functions` in code. They are a logical grouping of behaviors.

      Examples #

      Basic #

      A very basic example of a step might be to fetch the code for the current repo.

      - name: Checkout
  uses: actions/checkout@v3

      Steps may have configuration, the format and specifics of the configuration are dependent on action being used.

      - name: Checkout
  uses: actions/checkout@v3
  with:
    submodules: true
    fetch-depth: 0

      Steps may also be as simple as running a script:

      - name: Build
  run: hugo --minify

      Complex scripts may use the yaml multiline syntax to provide

      - name: Build
  run: |
    echo "doing thing"
    date
    sleep 1
    echo "done"
